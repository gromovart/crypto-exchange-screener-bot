# Пакет config

Пакет `config` предоставляет централизованную конфигурацию приложения crypto-exchange-screener-bot.

## Структура пакета

Пакет разбит на логические модули для улучшения читаемости и поддерживаемости:

### 📄 config.go (≈15 строк)
Главный файл пакета с документацией и описанием структуры.

### 📄 types.go (≈400 строк)
Определение всех структур конфигурации:
- `Config` - основная структура конфигурации
- `DatabaseConfig` - настройки PostgreSQL
- `RedisConfig` - настройки Redis
- `AnalyzerConfig` - конфигурация анализаторов
- `UserDefaultsConfig` - настройки пользователей по умолчанию
- И другие вложенные структуры

### 📄 loader.go (≈380 строк)
Загрузка конфигурации из переменных окружения:
- `LoadConfig(path string) (*Config, error)` - основная функция загрузки
- Чтение и парсинг .env файлов
- Установка значений по умолчанию
- Обработка обратной совместимости

### 📄 validators.go (≈120 строк)
Валидация конфигурации:
- `validate() error` - приватная функция валидации
- `Validate() error` - публичный метод валидации
- Проверка обязательных полей
- Проверка корректности значений

### 📄 helpers.go (≈115 строк)
Вспомогательные функции для чтения переменных окружения:
- `getEnv()` - чтение строковых значений
- `getEnvInt()` - чтение целых чисел
- `getEnvFloat()` - чтение чисел с плавающей точкой
- `getEnvBool()` - чтение булевых значений
- `getEnvDuration()` - чтение временных интервалов
- `parseIntList()` - парсинг списков целых чисел
- `parsePatterns()` - парсинг паттернов

### 📄 methods.go (≈330 строк)
Методы для работы с конфигурацией:

**База данных:**
- `GetDatabaseConfig()` - получение конфигурации БД
- `GetPostgresDSN()` - получение DSN для PostgreSQL

**Анализаторы:**
- `IsCounterAnalyzerEnabled()` - проверка включения анализатора
- `GetCounterAnalysisPeriod()` - получение периода анализа
- `GetCounterGrowthThreshold()` - порог роста
- `GetCounterFallThreshold()` - порог падения
- `GetEnabledAnalyzers()` - список включенных анализаторов

**Telegram:**
- `IsWebhookMode()` - проверка режима webhook
- `IsPollingMode()` - проверка режима polling
- `GetWebhookURL()` - получение URL webhook

**Redis:**
- `GetRedisAddress()` - получение адреса Redis
- `GetRedisPassword()` - получение пароля
- `GetRedisDB()` - получение номера БД
- `GetRedisPoolSize()` - размер пула соединений

**Общие:**
- `PrintSummary()` - вывод сводки конфигурации
- `IsDev()` - проверка режима разработки

## Использование

```go
import "path/to/internal/infrastructure/config"

// Загрузка конфигурации
cfg, err := config.LoadConfig("configs/prod/.env")
if err != nil {
    log.Fatal(err)
}

// Валидация
if err := cfg.Validate(); err != nil {
    log.Fatal(err)
}

// Вывод сводки
cfg.PrintSummary()

// Использование методов
dsn := cfg.GetPostgresDSN()
redisAddr := cfg.GetRedisAddress()
```

## Преимущества разделения

1. **Читаемость**: каждый файл отвечает за свою область
2. **Поддерживаемость**: легко найти нужную функциональность
3. **Тестируемость**: можно тестировать каждый модуль отдельно
4. **Расширяемость**: легко добавлять новую функциональность
5. **Навигация**: проще ориентироваться в коде
